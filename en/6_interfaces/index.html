Interfaces in Pyplan#

Introduction to interfaces#

In this section is where the input and output data tools of the applications created with Pyplan are created and managed. The interfaces are composed of components (widgets) that are dragged from a toolbox to the screen of the future interface. These components permanently communicate with the calculation rules to allow interaction between the user and the application. It is in this way that when modifying an input parameter, that alteration is computed in real time and the output result presented to the user.

To access the interfaces module, click on the section called “Interfaces” in the main menu.

Elements to include in an interface#

There are different components that can be inserted into an interface

Node#

This is the most general case. When we drag a node, it is presented in the interface with the same display configuration that was determined in the diagram. This view can later be configured differently and the changes will not affect the diagram view.

Index#

When an index is dragged, it will automatically be considered as a filtering interface for all other elements (graph or table) that have that dimension in their data structure. If this is not the desired behavior, you can indicate the non-synchronization of an element in relation to one or more indexes.

The first attribute to configure is whether the selector will be a single element or many:

Additionally, it is possible to choose some representation options of this selector according to the visualization needs:

Chart, Table or Indicator#

When dragging a graphic, table or indicator type component, the default view of the node is not considered (even if it is a graphic, table or indicator type). Once the component is located, you can start configuring it according to the desired behavior.

In addition to the style settings it is possible to define the link between the dimensions of a particular node and the explicit indices in that interface.

For example, in this case you can see how when you have de-selected the synchronization of the “Surface” index, when you click on it, no filtering is produced in the graph.

Events in charts#

In the charts configuration it is possible to activate some events to configure the behavior of other graphics.

There are three types of events:

  • Drilldown: double-clicking on a chart allows you to browse other dimensions that have been grouped together.

  • Selection event: by checking this box, the selection made on this chart is automatically applied to any other chart with shared dimensions.

  • Hover event: allows you to identify what is being pointed out in a chart and use it to act as a filter in another chart.
